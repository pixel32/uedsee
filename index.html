UX设计原则：冯·莱斯托夫效应

发表于 2021-05-25 评论：

UX设计原则：冯·莱斯托夫效应

摘要：相对于普通事物，记住独特或特殊事物的可能性更大。

关键词：UX设计，冯·雷斯托夫效应

概述

相对于普通事物，记住独特或特殊事物的可能性更大。

冯·莱斯托夫效应是给予一组事物中的独特项目更多注意的结果，这一组事物 可能是一列单词、一些物体、一系列事件、人名或面孔。

当情境中有差异（也即：一种不同于周围刺激的刺激）或经历中存在差异（也即：不同于记忆中经历的刺激）时，冯·莱斯托夫效应就产生了。

起源

该理论是由德国精神科医生和儿科医生赫德维格·冯·莱斯托夫（Hedwig von Restorff，1906–1962年）提出的，她在1933年的研究中发现，当向参与者展示一系列类别相似的物品时，与众不同的物品更容易被注意到。

应用

  • 使重要信息或关键操作在视觉上具有特色。
  • 在强调视觉元素时要克制，以避免它们相互竞争，并确保突出的项目不会被错误地识别为广告。
  • 不要仅仅依靠颜色来传达对比度，还要考虑那些有色觉不足或视力低下的人。
  • 使用动画传达对比度时，需要考虑对动画不敏感的用户。

UX设计原则：泰斯勒定律

发表于 2021-05-21 评论：

UX设计原则：泰斯勒定律

摘要：泰斯勒定律（又称“复杂性守恒定律”）指出，对于任何系统，都存在一定程度的复杂性，无法降低。

关键词：UX设计，泰斯勒定律

概述

泰斯勒定律（又称“复杂性守恒定律”）指出，对于任何系统，都存在一定程度的复杂性，无法降低。

起源

在1980年代中期为Xerox PARC工作时，拉里·特斯勒（Larry Tesler）意识到用户与应用程序交互的方式与应用程序本身同样重要。丹·萨弗（Dan Saffer）撰写的《互动设计》（Designing for Interaction）一书接受了拉里·特斯勒（Larry Tesler）的采访，描述了保持复杂性的规律。访谈在用户体验和交互设计师中很受欢迎。拉里·特斯勒（Larry Tesler）认为，在大多数情况下，工程师应该花额外的时间来减少应用程序的复杂性，而不是因为额外的复杂性而使数百万的用户花更多的时间使用该程序。但是，布鲁斯·托格纳西尼（Bruce Tognazzini）提出，人们应该减少生活中的复杂性。因此，当简化应用程序时，用户开始尝试更复杂的任务。

泰斯勒定律定律认为：无论在产品开发环节还是在用户与产品的交互环节，其内在的复杂度都有一个临界值，到达临界值后就不能再简化了，你唯一能做的就是将固有的复杂性从一个地方移动到另外一个地方。

应用

  • 所有过程都具有无法设计的复杂性核心，因此必须由系统或用户来承担。
  • 通过设计和开发过程中固有的复杂性，确保尽可能减轻用户的负担。
  • 注意不要将接口简化到抽象。

UX设计原则：系列位置效应

发表于 2021-05-19 评论：

UX设计原则：系列位置效应

摘要：用户最容易记住序列中的第一项和最后一项。

关键词：UX设计，系列位置效应

概述

用户最容易记住序列中的第一项和最后一项。

系列位置效应（英语：serial-position effect）是一种心理学现象，指人们倾向于对首先见到的事物和最后见到的事物有更好的印象。分为首因效应（primacy effect）和近因效应（recency effect）。

起源

系列位置效应是Herman Ebbinghaus创造的一个术语，描述了序列中某个项目的位置如何影响记忆准确性。指人们倾向于对首先见到的事物和最后见到的事物有更好的印象。分为首因效应（primacy effect）和近因效应（recency effect）。

首因效应是对“第一印象”或“先入为主”所产生的心理学现象，指在行为过程中，最先接触的事物会给人留下深刻的感知或认知，影响人对事物的感知和判断。

近因效应指在行为过程中，对事物的最近一次接触给人留下深刻的感知或认知，能对首因效应所形成的心理作用起到巩固、维持、否定、修改或调整等作用。例如在时间上与下一次购买决策最为接近，因此正或负的近因效应能促进或阻滞购买行为。

应用

  • 将最不重要的项目放在列表的中间可能会有所帮助，因为这些项目倾向于较少地频繁存储在长期记忆和工作记忆中。
  • 在诸如导航元素的最左侧和最右侧放置关键按钮可以增强记忆。

UX设计原则：波斯特尔定律

发表于 2021-05-15 评论：

UX设计原则：波斯特尔定律

摘要：你接受什么是自由的，而你发送什么是受限制的（保守的）

关键词：UX设计，波斯特尔定律

概述

波斯特尔法则：你接受什么是自由的，而你发送什么是受限制的（保守的）。

起源

互联网的先驱乔恩·波斯特尔（Jon Postel）制定了波斯特尔定律（也称为稳健性原则）。

该法律是互联网的设计指南，特别是关于TCP和网络的法律，并指出“TCP实现应遵循鲁棒性的一般原则：对自己的工作要保守，对他人的接受要自由”。换句话说，向其他计算机（或同一计算机上的其他程序）发送消息的程序应完全符合规范，但接收消息的程序只要含义清楚就应接受不一致的输入。

应用

  • 对用户可以采取的各种行动或他们可能提供的任何输入抱持灵活态度并容忍他们。
  • 提供可靠且可访问的界面同时，用户在输入，访问和功能方面进行何种操作都有可能。
  • 我们对设计的期望和计划越多，设计的弹性就越大。
  • 接受来自用户的可变输入，翻译该输入以满足系统的要求，定义输入边界，并向用户提供清晰的反馈。
